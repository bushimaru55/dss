## 文書目的

[SPEC-TI-001](SPEC-TI-001-table-read.md) の `TableReadArtifact` と [SPEC-TI-002](SPEC-TI-002-judgment.md) の `JudgmentResult`（および 002 が evidence 等で渡す**一次ラベル・見出し前提**）を入力とし、**後続の分析・保存・説明生成**に使える **内部正規表現**（論理行・列値・メタ）へ写像する**変換規約の正本**を定義する。

**明記（責務の正本）**

- **[SPEC-TI-001](SPEC-TI-001-table-read.md)** の読取結果は、**原本セル・座標系・結合の観測正本**である。003 は **001 の `cells`／`merges`／`parse_warnings` を上書きしない**。正規化は**参照**し、`trace_map` で**由来を失わない**。  
- **[SPEC-TI-002](SPEC-TI-002-judgment.md)** の判定結果は、`taxonomy_code`・見出し採否の前提・行種別・列役割・単位候補（および `decision`／`evidence`）の**入力前提**である。003 はこれを**解釈して変換する**が、**新たな表種別の確定や見出しの再判定**は行わない（矛盾は**保留フラグ**・`NEEDS_REVIEW` 連携で扱う）。  
- **本仕様（003）** は **変換・正規化アルゴリズムと `trace_map` 方針の正本**である。  
- **[SPEC-TI-004](../03_analysis_human/SPEC-TI-004-analysis-metadata.md)** の `dimensions`／`measures`／`grain` は、**003 の `NormalizedDataset` を前提**に**意味づけ・宣言**する（004 が最終の grain 意味正本）。  
- **[SPEC-TI-005](../03_analysis_human/SPEC-TI-005-human-review-flow.md)** へは、**正規化不能箇所・複数解釈箇所・部分正規化の範囲**を、**構造化して**渡せる出力前提を満たす（画面文言・ステップ詳細は 005／007）。  
- **[SPEC-TI-011](SPEC-TI-011-confidence-scoring.md)** へは、正規化段階の**不確実性**（欠落 binding、型パース失敗、部分スキップ等）を **011 が消費しうる形**で載せる（数式・閾値は 011 正本）。

## 用語定義

| 用語 | 説明 |
|------|------|
| 内部正規表現 | [SPEC-TI-006 §5.7](../01_foundation/SPEC-TI-006-io-data.md) の `NormalizedDataset`（`rows[]`＋`trace_map`）が担う論理表現。 |
| 論理行 | 正規化後の `rows[]` の 1 要素。元表の 1 明細行に相当する場合と、縦持ちにより **1 度量×1 交差** に相当する場合がある。 |
| 縦持ち | クロス表などから、**軸キー列＋度量列（および単位列）** への変換。複数交差セルが複数論理行になる。 |
| 見出し展開 | 多段見出しを、論理行の**ディメンション値**（または仮想列）へ落とし込む過程。010 の `row_path`／`column_path` に依存。 |
| 結合補完 | 結合矩形の**代表セル**以外へ、同一キャプション／値を**論理的に複製**する扱い（原本座標は `trace_map` で区別）。 |
| 部分正規化 | 表の一部ブロックのみ `rows[]` を生成し、残りを **skipped／INCOMPLETE** としてメタに残す状態。 |
| 正規化不能 | 入力契約を満たさない、または binding が再構成不能で **安全な論理行が生成できない** 状態。 |

## 正規化の位置づけ

表解析パイプラインにおいて、003 は次に限定する。

1. **読取の観測を尊重**: 001 のセル・結合・警告を**真実**として、捏造しない。  
2. **判定のラベルを消費**: 002 の `taxonomy_code`・行種別・列役割・単位スコープを**変換パラメータ**として使う。  
3. **構造に従う**: 010 の binding が利用可能なら、縦持ち・多段展開は **binding 優先**。未整備時は暫定ルール（§正規化不能時）。  
4. **追跡可能性**: 各論理フィールドが**どの原本セル（範囲）に由来するか**を `trace_map` で失わない。  
5. **ゲーティング連携**: `JudgmentResult.decision === NEEDS_REVIEW` でも**部分正規化を試みるか**は製品方針（暫定は §正規化不能時）。

**境界の再掲**: **001＝読取と一次構造化**、**002＝採否・分類・判定**、**003＝変換・正規化**。003 がこの境界を侵さないこと。

## 入力

### 必須

- **`TableReadArtifact`**（006 §5.4、[SPEC-TI-001](SPEC-TI-001-table-read.md) 準拠）  
  - `cells`, `merges`, `parse_warnings[]`  
- **`JudgmentResult`**（006 §5.6、[SPEC-TI-002](SPEC-TI-002-judgment.md) 準拠）  
  - `taxonomy_code`, `decision`, `evidence[]`（非空）  
- **`table_id`**（001／002 と同一）  
- **座標系**: [SPEC-TI-001](SPEC-TI-001-table-read.md) の **0-based inclusive**（`bbox`・セル座標）。`trace_map` もこの系で記述する。

### 推奨（あれば利用）

- **`HeadingTree`**（006 §5.5、[SPEC-TI-010](../01_foundation/SPEC-TI-010-heading-model.md)）：`cell_bindings` がある場合、縦持ち・多段展開の**主入力**。  
- **002 `evidence[].details`**: 行種別マップ、列役割マップ、単位候補、`UNIT_SCOPE_*`、`ambiguity` 等（006 MINOR または暫定 JSON）。

### 入力の優先順位（矛盾時・暫定）

1. **001 の観測**（セル値・結合）を**最優先**（002 の断定と矛盾する場合は **正規化結果に `conflict` メタ**を付し、005 連携）。  
2. **002 の `taxonomy_code`** は**単一ラベル**として経路選択に使う（副候補は evidence のみ）。  
3. **010 binding** が **001 と整合**する場合、binding を採用。整合しない場合は **INCOMPLETE** 扱い（§正規化不能時）。

## 出力

### 主成果物: `NormalizedDataset`

[SPEC-TI-006 §5.7](../01_foundation/SPEC-TI-006-io-data.md) の必須フィールドを満たす。

| フィールド | 003 での意味（暫定） |
|------------|------------------------|
| `dataset_id` | 正規化実行の一意 ID。 |
| `table_id` | 入力と同一。 |
| `rows[]` | 論理行の配列。各要素の**概念上のまとまり**は下記「`rows[]` の論理要素」を参照（物理キー名は 006 Phase4 で固定）。 |
| `trace_map` | **論理フィールド → 原本セル（または範囲）** の写像。§trace_map 参照。 |

### `rows[]` の論理要素（概念・Draft 0.2）

厳密な JSON Schema は 006 Phase4 に委譲する。004／015 が同じ前提で接続できるよう、**1 論理行**に次の区分を**概念として**載せうる（実装はフラット辞書でもネストでも可。意味の束ねは以下）。

| 区分 | 意味（暫定） | 004 への含意 |
|------|----------------|--------------|
| **key fields** | 行を識別・分割する軸（多段見出し由来の属性値、KEY_VALUE のキー、CROSSTAB 縦持ち後の行軸・列軸キー等）。 | **dimension 候補**の主たる入力。 |
| **value fields** | 分析対象の度量・カテゴリ値（セル実値の正規化結果）。 | **measure 候補**または事実テーブルの値。 |
| **metadata fields** | 行種別タグ、単位タグ、パース失敗フラグ、002 由来の列役割タグ、正規化バージョン等（**原本値の代替ではない**）。 | 004 の制約・警告・派生列設計の入力。 |
| **source trace reference** | 各 key／value（必要なら metadata）が **`trace_map` のどのキーと対応するか**の参照（論理フィールド ID 列挙または埋め込みポインタ。暫定）。 | 説明・監査で「この値の出所」を 001 座標へ戻す。 |

**明記**: **grain の宣言**や「どれを dimension／measure と呼ぶか」の**最終命名**は [SPEC-TI-004](../03_analysis_human/SPEC-TI-004-analysis-metadata.md) の責務。003 は上記区分を**揃えて出力**し、004 がそのままマッピングできる**材料**を渡す。

### 副次成果物（006 MINOR または `rows[]`／`trace_map` 内メタ）

初版では **006 を壊さない**ため、次を **拡張メタ**（例: `normalization_meta` オブジェクト）として載せうる。

- **`normalization_status`**: `COMPLETE` \| `PARTIAL` \| `FAILED`（暫定 enum）  
- **`skipped_regions[]`**: 正規化対象外にした矩形・行範囲（0-based inclusive）と理由コード（**消去ではなく出力先の分離**。§trace_map 参照）  
- **`aggregate_rows[]`**（概念）: 集計行を `rows[]` から外したときの**ラベル・値・元行参照**（§集計行・注記行・見出し帯）  
- **`note_blocks[]`**（概念）: 注記行のテキストと**原本参照**（§集計行・注記行・見出し帯）  
- **`incomplete_bindings[]`**: 010 の INCOMPLETE に対応するデータセル参照  
- **`type_normalization_notes[]`**: 数値・日付パースの失敗・丸め（011 入力用）  
- **`unit_application[]`**: 論理列または論理行への単位の付与結果（`UNIT_SCOPE_*` 由来）

## 後続（004／005／011）への出力前提

- **004**: `rows[]`（key／value／metadata／trace 参照の概念区分）と `trace_map` を入力とし、**dimensions／measures／grain** を**宣言**する。003 は **004 の意味づけを代替しない**（004 正本）。多段見出しの**列の並び**は「LIST_DETAIL における多段見出しの展開（Draft 0.2 暫定標準）」を前提に書ける。  
- **005**: `normalization_status` が `PARTIAL`／`FAILED`、または `incomplete_bindings`・`skipped_regions`・`aggregate_rows[]`・`note_blocks[]` が非空のとき、**人確認キュー**に載せうる**構造**を持つ（§正規化不能時・§trace_map）。質問文面は 005。  
- **011**: `type_normalization_notes`・binding 欠落・`PARTIAL`／`FAILED`・部分スキップの**件数・比率**を**特徴量**として渡す（重み付けは 011）。状態の**意味**は本書 §正規化不能時 を参照。

## 正規化の原則

1. **非破壊**: 001 Artifact を書き換えない。  
2. **非判定**: 002 の taxonomy／見出し採否を**覆さない**。矛盾はメタに記録。  
3. **座標一貫**: 全 trace は **001 の 0-based inclusive**。  
4. **説明責任**: 主要な論理値は **`trace_map` にエントリ**を持つ（集計行を除外した場合も、**除外理由と元行**の参照を残す方針）。  
5. **タイプ別経路**: [SPEC-TI-009 §7](../01_foundation/SPEC-TI-009-table-taxonomy.md) のマトリクスに沿い、**主経路**を選択（詳細アルゴリズムは本書の方針レベルに留める）。  
6. **無理な埋め合わせ禁止**: 欠落見出しを**確定値で補う**のではなく、`is_guess`／INCOMPLETE を 010／002 の前提に合わせて伝播する。

## 表種別（taxonomy）ごとの正規化方針の差（初版）

方針は [SPEC-TI-009 §6–7](../01_foundation/SPEC-TI-009-table-taxonomy.md) と整合する。**同一タイプ内でもレイアウトで分岐**しうるため、以下は**初版の主経路**であり、例外は §正規化不能時へ回す。

| taxonomy_code | 主経路（003） | 備考（初版） |
|---------------|----------------|----------------|
| `TI_TABLE_LIST_DETAIL` | **ワイド行の保持**＋型正規化。多段見出しはキー列へ展開。 | クロス展開は原則不要。集計行は §集計行・注記行・見出し帯。 |

| `TI_TABLE_KEY_VALUE` | キー列・値列から **1 論理行** または **縦長キー値**へ正規化。分岐は §KEY_VALUE の経路分岐。 | taxonomy は再判定しない。 |

| `TI_TABLE_UNKNOWN` | **正規化保留**または**最小限のセルフラット化**（暫定）。 | `decision`・evidence に従い、005 優先。§正規化不能時（UNKNOWN と `normalization_status`）。 |

## KEY_VALUE の経路分岐（Draft 0.2 暫定）

`TI_TABLE_KEY_VALUE` について、**taxonomy の再判定は行わず**、レイアウトと反復パターンから **003 の変換経路**だけを選ぶ。

| 条件 | 優先経路 |
|------|-----------|
| **単一レコード**の属性羅列（キー種が一巡し、同種キーの繰り返しがない） | **1 論理行**（ワイド化）を**優先**。key fields に各属性、value fields に対応値。 |
| **同一キー種別が縦に繰り返される**（同型の「項目／値」ペアが複数行） | **縦長キー値形式**を**優先**（1 キー列＋1 値列＋論理行複数）。 |
| **帳票断片・セクションブロックが反復**（FORM_REPORT 寄りだが KEY_VALUE と判定済みの場合を含む） | **セクション単位でブロック保持**、または **部分正規化**（ブロックごと `rows[]`、境界は `skipped_regions`／メタ）。 |
| **上記が競合し曖昧** | **`normalization_status=PARTIAL`** を付与し、**005／review** へ逃がす前提。無理に単一路線に確定しない。 |

## 時系列表（TIME_SERIES）のワイド保持と縦持ち（Draft 0.2 暫定）

**完全なアルゴリズムは書かない**。004 が grain・time 意味を書くための **003 の経路選択基準**のみ。

| 条件 | 暫定方針 |
|------|-----------|
| 時系列列が**連続**し、**measure 系列が単純**（主に数値列が時系列に沿って並ぶ） | **縦持ちを優先**（時間を key field、度量を value field とした論理行を複数生成）。 |
| **固定比較列**（識別子・カテゴリ）を横に残し、時間だけを崩すと解釈が損なわれる場合 | **ワイド保持を許容**（時間列は横のまま、**時間メタ**・列役割タグを付与）。 |
| 002 で **`COL_TIME` が複数の連続列**として付いている | **縦持ち候補を強める**（列方向の時間パターンが明確なため）。 |

**維持**: **暦・会計期間・週次などの最終意味づけ**は **004 に委譲**。003 はパース可能なら型正規化し、`metadata` に曖昧性を残しうる。

## 一覧表の正規化方針

- **対象**: 主に `TI_TABLE_LIST_DETAIL`、および `TI_TABLE_TIME_SERIES` のワイド保持モード。  
- **行**: `ROW_DETAIL` に相当する行のみを **データ `rows[]` の母集団**とする（集計・注記・見出し帯は §集計行・注記行・見出し帯）。  
- **列（多段見出し）**: §LIST_DETAIL における多段見出しの展開（Draft 0.2 暫定標準）に従う。  
- **型**: 各列について §数値・日付・文字列 を適用。列単位の型は**多数決＋例外セルメタ**（暫定）。  
- **trace_map**: 各論理セル値 → **元の 1 セル**（結合ならアンカー＋範囲メタ）。

### LIST_DETAIL における多段見出しの展開（Draft 0.2 暫定標準）

004 が dimensions／measures を**同じ前提**で組み立てられるよう、LIST_DETAIL（ワイド保持）での多段見出しを次で固定する（JSON Schema 固定はしない）。

- **既定推奨**: **個別列分離**（外側 level → 内側 level を **別々の key fields／論理列** に対応させる）。010 の `column_path`／`row_path` の**各ノードが 1 列（または 1 組の key）**に写像しやすく、004 の dimension 割当が直線的になる。  
- **連結キー（例外的）**: 列数上限・下流互換などで**単一キー列**にまとめる場合のみ許容。**区切り方針（暫定）**: 外側→内側の順に、**同一の区切り文字**で連結する（例: ` › `。実装・製品で統一。スラッシュ `/` は日付・分数と誤認しやすいため**非推奨**）。  
- **欠落階層・空 caption**: **列を捏造しない**。該当 level の値は **NULL／空**を許容し、**metadata** に「欠落」「空見出し」を記録。010 で `INCOMPLETE` のノードは §多段見出し表の展開どおり **論理行を作らない**か **PARTIAL** とする。  
- **004 への明記**: 003 の出力（個別列または連結列＋欠落メタ）を**前提**に、004 は **dimensions の個数・階層対応**と **measure 列**を宣言する。003 は **dimension 名のビジネス用語**を確定しない。

## クロス集計表の正規化方針

- **対象**: `TI_TABLE_CROSSTAB`（行軸・列軸・交差度量）。  
- **縦持ち 1 論理行**: 少なくとも **行軸の合成キー**、**列軸の合成キー**、**度量値** を含む（単位は §単位候補の反映方針）。  
- **入力**: 010 の `cell_bindings` を**優先**。各交差セルについて `row_path`／`column_path` からキャプションを解決する。  
- **空セル**: スパース表では **論理行を生成しない**か、**NULL 明示行**を生成するかは製品方針（暫定: **値のあるセルのみ**行生成し、解析側で外挿しない）。  
- **trace_map**: 論理フィールドごとに、**対応する見出しセル**と**交差セル**の両方を参照しうる（配列で保持）。

## 多段見出し表の展開方針

- **前提**: [SPEC-TI-010](../01_foundation/SPEC-TI-010-heading-model.md) の **level**（外側ほど小）と **継承**に従う。  
- **展開**: LIST_DETAIL の**暫定標準**は §LIST_DETAIL における多段見出しの展開。CROSSTAB／縦持ちでは、パス各段を **key fields**（行軸・列軸の複数列または連結）へ写像する。  
- **欠落**: `INCOMPLETE` binding は論理行を生成せず、**`incomplete_bindings`** に列挙。  
- **疑似見出し**: 002 が不採用としたノードは **展開対象に含めない**（002 前提の尊重）。

## 結合セルの補完方針

- **正本**: 001 の `merges` と、[SPEC-TI-001](SPEC-TI-001-table-read.md) の **アンカー／代表セル**規約。  
- **補完**: 結合領域内の「従属セル」は、正規化時に**代表値と同一の論理値**を持ちうる。従属セル側にも `trace_map` を**別エントリ**で持ち、**同一 `merge_id` メタ**でグルーピング可能にする（暫定）。  
- **見出し結合**: 010 ノード 1 矩形と一致させ、**データセルへの継承**は binding に従う。  
- **T 字・L 字**: 親子矛盾は 002／010 段階で検知済みを前提。003 で検知した場合は **PARTIAL**＋理由。

## 行種別の反映方針

| 行種別 | 003 の扱い（暫定） |
|--------|---------------------|
| `ROW_DETAIL` | **`rows[]` の通常データ行**として出力。 |

| `ROW_NOTE` | **原則 `rows[]` に含めない**。**注記テキスト**は `skipped_regions` または `note_blocks[]` に保持（005・説明生成向け）。 |
| `ROW_HEADER_BAND` | **データ行化しない**。見出し展開の入力のみ。 |
| `ROW_UNKNOWN` | **除外**または**1 行だけ試験的に含む**は `decision` 連動（暫定: **除外＋メタ**）。 |

**明記**: **集計行を残すか除外するか**のデフォルトは上表。**最終分析方針**は 004／製品と整合させる。

## 列役割の反映方針

002 の列役割（`COL_ATTRIBUTE`／`COL_MEASURE`／`COL_TIME`／`COL_UNIT`／`COL_NOTE`／`COL_UNKNOWN`）を、論理列に**タグ**として付与する（物理フィールド名は 006）。

- **`COL_ATTRIBUTE`**: ディメンション**候補**として `rows[]` のキー列に対応。  
- **`COL_MEASURE`**: 度量**候補**。縦持ち後は **measure 名＋値**の列ペアに再配置しうる。  
- **`COL_TIME`**: 時間軸**候補**。パースは §数値・日付・文字列。意味は 004。  
- **`COL_UNIT`**: 単位が列全体に付く場合のヒント（§単位と併用）。  
- **`COL_NOTE`**: データ本体から**分離**し、注記メタへ。  
- **`COL_UNKNOWN`**: 004 で再分類。**003 は無理に型を確定しない**。

## 単位候補の反映方針

| 仮ラベル | 003 の反映 |
|----------|------------|
| `UNIT_SCOPE_TABLE` | **全論理行**または**全度量列**に共通単位メタを付与。`trace_map` は単位元セル（行）を指す。 |
| `UNIT_SCOPE_COLUMN` | **対象列に属する論理フィールド**に単位を付与。 |
| `UNIT_SCOPE_CELLS` | **セル群の bbox** に一致する論理値のみに単位を付与。 |

## 数値・日付・文字列の正規化方針

- **数値**: カンマ区切り・括弧負数・通貨記号の**除去ルール**はlocale方針に従う（詳細は実装／012）。**パース失敗**は元文字列を保持し、**失敗フラグ**をメタに載せる（011 入力）。  
- **日付**: ISO 8601 への正規化を**推奨**。曖昧な日付（2桁年など）は **原文保持＋警告**。  
- **文字列**: 前後空白除去、改行の正規化（LF）を**推奨**。全角半角の統一は**列ごとオプション**（暫定）。  
- **確定閾値**（「この程度なら数値確定」等）は **011／製品**。**003 はルールベースの試行**に留める。

## 集計行・注記行・見出し帯の扱い

- **集計行**: デフォルトでは **`rows[]` から分離**し、**小計／合計のラベルと値**を **`aggregate_rows[]`（概念）** に残す（ピボット再構成や説明生成用）。**消去しない**。詳細は §trace_map。  
- **注記行**: **`rows[]` に含めない**。**`note_blocks[]`（概念）** にテキストと**原本参照**を載せる。正規化対象の「データ行」との**境界**は 002 前提を尊重。  
- **見出し帯**: **データ行化しない**。見出し展開の入力のみ。帯内セルは `trace_map` の**キー側**にのみ現れる。

## trace_map／原本参照方針

- **原本の正本**: [SPEC-TI-001](SPEC-TI-001-table-read.md) のセル座標（**0-based inclusive**）および `merges`。  
- **目的**: 正規化後の**各論理フィールド**が、**どの原本セル（または矩形）に由来するか**を機械的に辿れること。  
- **思想**: 除外・未採用は**原本情報を消さない**。**`rows[]` に載せない＝捨てる**ではなく、**出力先を分ける**（005・説明責任向け）。  
- **粒度（暫定）**: **セル単位**を最小。同一値が結合補完で複製される場合は**複数セル参照**を許容。  
- **必須性**: 006 で `trace_map` が必須のため、**少なくとも `rows[]` に現れる全値**についてエントリを持つことを**初版成立ライン**とする（中間計算用の内部キーは省略可）。  
- **警告連携**: 001 `parse_warnings` との接続は **参照 ID** または **セル座標**で行い、003 は warnings を**改変しない**。

### `rows[]` とメタ出力先の対応（Draft 0.2）

| 対象 | `rows[]` | その他の出力先 | trace の扱い（原則） |
|------|----------|----------------|----------------------|
| 明細（`ROW_DETAIL`） | **含める** | — | 各 value／key について **データセル**（＋見出し由来 key は**見出しセル**）へ `trace_map` |
| 集計行（除外時） | **含めない**（デフォルト） | **`aggregate_rows[]`** | 集計セル（値）と**ラベルセル**を **`trace_map` または aggregate 専用 trace スロット**に残す（006 MINOR で一本化可） |
| 注記行 | **含めない** | **`note_blocks[]`** または **`skipped_regions[]`**（理由 `NOTE`） | 注記を構成するセル範囲を**参照として保持**し、005 が原本を表示しうる |
| 見出し帯 | **含めない** | 見出し展開の入力のみ | **キー側**の `trace_map` のみ（データ値としては重複登録しない） |
| binding 欠落で**未採用**のデータセル／範囲 | **当該論理行は生成しない** | **`incomplete_bindings[]`** | **未採用であること**と**元セル座標**を列挙。必要なら `skipped_regions` と併記 |

## 正規化不能時の扱い

### `normalization_status` の境界（Draft 0.2 暫定）

| 状態 | 意味（判断基準） |
|------|-------------------|
| **`PARTIAL`** | **一部**の論理行、または **`rows[]` 内の一部論理フィールド**について、**安全に**（001／002／010 の前提を大きく破らずに）生成できる。残りは `incomplete_bindings`・`skipped_regions`・曖昧メタに分離。 |
| **`FAILED`** | **安全に生成できる論理行がほぼない**（全体が binding 不能・入力破綻に近い、または REJECT 相当で空出力）。 |
| **`COMPLETE`** | 主要データ領域について論理行と trace が意図どおり揃い、致命的欠落がない（暫定）。 |

### 状況別の対応

| 状況 | 暫定方針 |
|------|-----------|
| `JudgmentResult.decision === REJECT` | **正規化スキップ**または**空に近い `rows[]`**。`normalization_status=FAILED`（またはジョブ側で未実行）。 |
| 必須 binding 欠落・CROSSTAB で軸が再構成不能 | 一部だけ出力可能なら **`PARTIAL`**。ほぼ出力不能なら **`FAILED`**。`incomplete_bindings` を充填。 |
| `TI_TABLE_UNKNOWN`＋**最小フラット化** | **暫定: `PARTIAL` 扱い**（セル値のフラット列挙のみで key 構造が弱いため）。完全スキップに近い場合は **`FAILED`**。005 へ。 |
| 002 `NEEDS_REVIEW` | **部分正規化を許可**（暫定: **許可**）。`PARTIAL`＋不確実性メタ。 |

**原則**: **無理に単一解釈へ収束しない**。複数解釈が残る場合は **`ambiguity` を `normalization_meta` に伝播**し、005 で解消。

### 005／011 から見た状態（参考）

- **005**: `PARTIAL` は「**直せる／選べる単位**が残っている」ことを示すメタとセットで渡す。`FAILED` は**再読取・再判定**が先か、**スキップ**かをフローで分岐（詳細は 005）。  
- **011**: `PARTIAL` は**減点要因**としつつ「一部確からしい」情報があること。`FAILED` は**高い不確実性**またはスコア対象外（閾値は 011）。
