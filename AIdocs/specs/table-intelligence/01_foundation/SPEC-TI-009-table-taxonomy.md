## 1. 目的とスコープ

### 目的

表読取・判定・正規化・分析メタデータ生成の **分岐根拠** となる、表タイプの **定義語彙（タクソノミー）** と各タイプに対する **後続処理方針** を単一の正本として定義する。

### スコープ（やる）

- 表タイプの列挙、各タイプの定義・典型例・非典型例の指針
- 分類ラベルと後続パイプライン（読取オプション、判定ルールセット、正規化経路の選択）の対応（マトリクス）
- 分類ラベルの **enum 名**（機械可読）と **利用者向け短名** の対応（[用語集-glossary.md](../../../00_管理/用語集-glossary.md) へ反映）

### スコープ外（やらない）

- セル格子の抽出アルゴリズム（SPEC-TI-001）
- 見出しセルの機械判定ルールの詳細（SPEC-TI-002／010 と分担）
- 信頼度の数値・閾値（SPEC-TI-011）

## 2. 関係仕様

| 仕様 | 関係 |
|------|------|
| SPEC-TI-010 | 見出しモデル。分類は見出し構造の前提を変えないが、タイプにより期待する見出しパターンが異なる。 |
| SPEC-TI-001 | 読取。分類に必要な **最小構造**（矩形・連結成分等）は 001 で定義し、009 はその上の **意味ラベル** を定義する。 |
| SPEC-TI-002 | 判定。分類の確定／保留は 002＋011 と連携。 |
| SPEC-TI-003 | 正規化。タイプごとの正規化経路を本書のマトリクスで参照する。 |
| SPEC-TI-006 | 入出力。`table_taxonomy_code` 等の列挙値の正本は本書＋006。 |
| SPEC-TI-008 | テスト。分類ラベルごとの必須フィクスチャを索引する。 |

**禁止越境**: 009 に HTTP や DB 物理名を書かない（006／014／015）。

## 3. 用語定義

| 用語 | 説明 |
|------|------|
| 表タイプ（table type） | 本書で定義する分類ラベル。1表に対し **single-label を既定**（multi-label は §10 で検討）。 |
| 典型例 | そのタイプの教師データ・説明用に用いる代表レイアウト。 |
| 非典型例 | 境界に近いが当該タイプに含めるケース、または誤分類リスクが高いケース。 |
| 後続処理方針 | 読取オプション、判定・正規化の **推奨経路**（必須ではなく 002／011 で最終ゲート）。 |

---

## 4. 入力・前提

- 入力は原則 **SPEC-TI-001 の TableReadArtifact**（矩形候補、セル格子、書式・結合のメタ）および **人確認で補正された構造**。
- 分類は **完全自動** と **人の選択** の両方を許容するが、列挙値の集合は本書で固定する。

## 6. 分類定義（各タイプ）

本版（Draft 0.1）では **8 分類** を置く。コードは `TI_TABLE_*` 接頭辞（006 でフィールド名に写像）。

### TI_TABLE_LIST_DETAIL（一覧明細型）

- **定義**: 行が観測単位（レコード）、列が属性・指標の **縦長の業務表**。最左列または最上行に見出しがあり、集計行が混在しうる。
- **典型**: 売上明細、在庫一覧、顧客リスト。
- **非典型**: 多段見出しのみが複雑で本体は一覧。
- **後続**: 正規化は主に **型正規化・集計行タグ付け**（003）。クロス展開は不要なことが多い。

### TI_TABLE_CROSSTAB（クロス集計型）

- **定義**: 行軸・列軸の両方に意味があり、交差セルが **度量** の **マトリクス表**。
- **典型**: 期間×品目の売上マトリクス。
- **非典型**: 行見出しのみ二段で列は単段。
- **後続**: **縦持ち展開** が主経路（003）。grain は 004 で宣言。

### TI_TABLE_FORM_REPORT（帳票・体裁型）

- **定義**: 印刷体裁・ラベル・空欄が主で、**格子としての規則性が弱い** または **複数ブロック** が混在。OCR／レイアウト解釈の比重が高い。
- **典型**: 申請書、政府・業界様式、複合フォーム。
- **非典型**: 一覧に近いが罫線が不規則。
- **後続**: 読取の **ブロック分割**（001）と人確認（005）を強く前提。

### TI_TABLE_PIVOT_LIKE（ピボット近似型）

- **定義**: Excel ピボットテーブルに近い **小計・合計行・階層折りたたみ** を含む表。一覧とクロスの中間。
- **典型**: ピボット出力、集計行付き多段見出し表。
- **非典型**: 単純一覧だが小計行のみ付与。
- **後続**: 集計行の **メタ分離**（002／003 の方針に依存）を要検討。

### TI_TABLE_TIME_SERIES（時系列横持ち型）

- **定義**: **時間軸が列方向** に並ぶ横持ち系列（月次列など）。一覧の特化形。
- **典型**: 月次推移表、累計列付き実績表。
- **非典型**: 四半期と月が混在。
- **後続**: 004 の `time` 解釈と 003 の **縦持ち** が密接。

### TI_TABLE_LOOKUP_MATRIX（照合・マトリクス型）

- **定義**: セルが **カテゴリラベルまたは記号** で、数値度量が主でない **照合・採番・マッピング** 用途。
- **典型**: 担当割当表、記号凡例付きマトリクス。
- **非典型**: 数値が一部だけ存在。
- **後続**: 分析メタ（004）は **measure 不在** になりやすい → 013 で候補抑制。

### TI_TABLE_KEY_VALUE（キー・値ブロック型）

- **定義**: 1列がキー、隣接列が値の **縦並びラベル表**。レコードが横に伸びない。
- **典型**: 設定一覧、フォームの「項目：値」列。
- **非典型**: キー列が2段に見える結合セル。
- **後続**: 正規化は **1レコード化**（ワイド化）または **エンティティ抽出**（003／004）。

### TI_TABLE_UNKNOWN（未分類・保留）

- **定義**: 自動分類の信頼度不足、または **複合タイプ** で単一ラベルが付かない暫定状態。
- **後続**: 005 人確認、または 002 での **再試行**。

## 7. 分類→後続処理マトリクス

| taxonomy_code | 読取（001）重点 | 判定（002）重点 | 正規化（003）主経路 |
|---------------|-----------------|-----------------|---------------------|
| LIST_DETAIL | 集計行検出 | 列意味・grain | 型正規化・集計行タグ |
| CROSSTAB | 軸セル特定 | 軸と度量の分離 | 縦持ち展開 |
| FORM_REPORT | ブロック分割 | レイアウト信頼度 | セクション単位正規化 |
| PIVOT_LIKE | 小計行・階層 | 集計行の扱い | 一覧化＋メタ保持 |
| TIME_SERIES | 期間列パース | 時系列軸 | 縦持ち（時間キー生成） |
| LOOKUP_MATRIX | 記号・空セル | 度量の有無 | スパース長形式 |
| KEY_VALUE | キー列安定化 | キー重複 | ワイド化／1行化 |
| UNKNOWN | — | 人確認へ | 保留 |

（表内は方針。実装は 006 のステータスと 011 の閾値と整合させる。）

---

## 8. 例外・境界事例

- **一覧とクロスの中間**: 既定は **PIVOT_LIKE** または **LIST_DETAIL** のどちらかに寄せるルールを 002 で確定（本書は「候補タイプ」を列挙するのみ）。
- **1シート複数表**: 各 **TableCandidate** ごとに分類（006）。シート単位ラベルは持たない。
- **multi-label**: 現状 **非採用**。必要なら MAJOR で `primary_code` + `secondary_codes[]` を 006 に導入。

---

## 9. テスト観点

- 各 `TI_TABLE_*` について **最低1件** のゴールデンファイル（001 出力＋期待 taxonomy）。
- **境界ペア**: LIST_DETAIL vs CROSSTAB、FORM_REPORT vs LIST_DETAIL、TIME_SERIES vs LIST_DETAIL。
- 列挙値の **typo 耐性**（API・DB は enum 固定、文字列自由入力を許さない）。

## 10. 未確定事項（暫定案）

| 論点 | 暫定案 |
|------|--------|
| single-label vs multi-label | **single-label 既定**。複合は UNKNOWN または人選択。 |
| 一覧と帳票の境界 | 規則的格子＋主に数値セル → 一覧寄り。不規則ブロック → 帳票。 |
| 分類の版管理 | `taxonomy_schema_version` を 006 で保持し、009 の MINOR と対応付ける。 |
